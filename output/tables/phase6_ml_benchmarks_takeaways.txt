Phase 6 ML and Non-Parametric Benchmarks - Capstone Takeaways

Core meaning:
Phase 6 answers supervisor-facing robustness questions about whether modern
machine-learning or non-parametric MIDAS methods can beat the main MIDAS
benchmarks. XGBoost, Kernel U-MIDAS, and LSTM have now been implemented.

1. XGBoost
- XGBoost RMSE: 0.02365 (-20.8% vs ARIMAX).
- Directional accuracy: 72.9%.
- XGBoost beats ARIMAX, so nonlinear machine learning does add value.
- XGBoost does not beat MIDAS nbeta (0.02057) or MIDAS nealmon (0.02104).
- Feature importance ranks prior-month WTI weeks 2-3 among the most important predictors, confirming the same 5-7 week transmission window found by MIDAS, CLM-SS, and LASSO-MIDAS.

2. Kernel U-MIDAS
- Implemented as a smooth non-parametric lag-weight model using a second-difference roughness penalty on the 12 weekly WTI lag coefficients.
- Lambda was selected on the 2013-2014 validation window.
- Best lambda: 0.000000, so validation chose no smoothing.
- Kernel U-MIDAS therefore collapses to unrestricted U-MIDAS in this dataset.
- Kernel U-MIDAS RMSE: 0.02226 (-25.5% vs ARIMAX), identical to U-MIDAS.
- It beats ARIMAX and XGBoost, but remains behind MIDAS nbeta and MIDAS nealmon.
- The estimated lag weights still peak at prior-month weeks 2-3.

3. LSTM
- Implemented in R using torch, with the torch backend installed successfully.
- Input: the same 12 weekly WTI lag sequence used by the MIDAS-style models.
- Output: one-step-ahead monthly CPI Energy log-change.
- Best validation setup: hidden size 4, 150 epochs, learning rate 0.010.
- LSTM RMSE: 0.02588 (-13.3% vs ARIMAX).
- LSTM MAE: 0.01840.
- LSTM directional accuracy: 76.0%.
- LSTM beats ARIMAX, so a recurrent neural network does extract useful signal from weekly WTI timing.
- LSTM does not beat MIDAS nbeta, MIDAS nealmon, U-MIDAS, Kernel U-MIDAS, or XGBoost.
- The likely reason is sample size: the capstone has only 275 monthly observations, with 96 rolling test forecasts. That is small for deep learning but acceptable for parsimonious MIDAS lag-shape models.
- Defense answer: yes, we tried deep learning in R; it improves over a monthly ARIMAX baseline but adds complexity without improving the main MIDAS result.

4. Performance vs interpretability table
- Final table saved to output/tables/performance_interpretability_table.csv and .md.
- Best RMSE: MIDAS nbeta, RMSE 0.02057 (-31.1% vs ARIMAX).
- Best directional accuracy: MIDAS nealmon, 80.2%.
- Best accuracy/interpretable trade-off: parametric MIDAS, especially nbeta and nealmon.
- LASSO-MIDAS is almost tied with MIDAS nealmon and independently selects the same prior-month lag window.
- XGBoost and LSTM both beat ARIMAX, but neither beats parametric MIDAS.
- Final defense line: the best model is not the most complex model; it is the model that preserves weekly timing while imposing enough structure to remain stable and interpretable in a small monthly macro-energy sample.

Main thesis takeaway:
Trying non-parametric and ML benchmarks strengthens the main capstone result.
XGBoost, Kernel U-MIDAS, and LSTM all beat ARIMAX, confirming that weekly WTI
timing contains useful information. However, none of these benchmarks beats
parametric MIDAS. Kernel U-MIDAS shows that the data do not require additional
smoothness beyond U-MIDAS, XGBoost confirms the lag hump through feature
importance, and LSTM shows that a deep recurrent model is not rewarded in this
small monthly sample. MIDAS therefore remains the best balance of accuracy,
interpretability, and economic lag-structure insight.
